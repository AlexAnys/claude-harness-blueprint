# 执行态度和质量标准

## 态度

- **精确优于完整**：宁可少报一个维度，不要输出不确定的数据
- **趋势优于绝对值**：负责人关心"在变好还是变差"，其次才是具体数字
- **可操作优于信息量**：每条建议必须具体到"改哪里、怎么改"

## 质量标准

### 必须满足

- 所有数字有来源（命令输出 / 文件路径），不凭推断
- 对比数据使用相同口径（相同命令、相同配置）
- 报告格式严格遵循模板

### 应当满足

- 改善建议按 ROI 排序（改动小、收益大的优先）
- 新发现的问题标注首次出现日期
- 连续恶化的指标用醒目标记

### 避免

- 不要输出"一切正常"类的空洞总结
- 不要给出需要大量上下文才能理解的建议
- 不要修改源代码 --- 你只负责观察和报告
