# 任务定义：每周代码质量报告

## 来源

项目技术负责人需要每周一了解代码仓库的健康状况，包括：
- 测试覆盖率趋势
- 新增 lint 警告
- 依赖安全漏洞
- 技术债变化

## 调度

- **频率**：每周一 09:00
- **触发方式**：cron 定时 / 手动执行
- **超时**：30 分钟
- **重试**：失败后 15 分钟重试一次，最多 2 次

## 目标

生成一份结构化的周报，包含：
1. 代码质量评分（0-10）
2. 与上周的对比
3. Top 3 改善建议
4. 需要立即处理的阻塞项

## 执行步骤

1. 读取 `context/tools.md` 获取可用工具和端点
2. 运行测试套件，收集覆盖率数据
3. 运行 linter，统计新增/修复的警告数
4. 检查依赖安全报告
5. 读取 `context/memory.md` 获取上周基线
6. 对比计算趋势
7. 生成报告到 `runs/YYYY-MM-DD.md`
8. 更新 `context/memory.md` 中的基线数据

## 成功标准

- [ ] 报告文件 `runs/YYYY-MM-DD.md` 已生成
- [ ] 报告包含所有 4 个维度的数据
- [ ] 与上周对比数据存在（非首次执行时）
- [ ] 评分有依据，不是凭感觉

## 失败处理

| 失败场景 | 处理方式 |
|----------|----------|
| 测试套件运行失败 | 记录失败原因，其他维度继续 |
| 无法获取上周基线 | 标记为"首次执行"，只出绝对值 |
| 超时 | 输出已完成部分，标记未完成项 |
| 依赖扫描工具不可用 | 跳过该维度，在报告中注明 |
