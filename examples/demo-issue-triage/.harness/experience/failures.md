# 故障记录

## Failure: 2026-04-28 | 安全漏洞误判为 P2

- **触发场景**：Issue #312 报告了 SQL 注入漏洞，但措辞较温和（"可能存在注入风险"）
- **错误行为**：Executor 分类为 bug/P2，标记为"有 workaround"
- **正确行为**：应为 P0 --- 任何安全漏洞默认 P0
- **根因**：优先级规则中"安全漏洞"的关键词匹配不够宽泛，缺少"注入"、"injection"、"CVE"
- **修正方案**：在分类规则中扩展安全相关关键词列表
- **是否已纳入 pattern**：是 --- 新增 Security-vulnerability pattern

## Failure: 2026-05-02 | Feature 请求被标记为 bug

- **触发场景**：Issue #330 标题为"JSON 字段排序不一致"，用户实际在请求"支持 JSON 字段排序"
- **错误行为**：关键词"不一致"触发了 bug 分类
- **正确行为**：应为 feature --- 用户在请求新功能
- **根因**：仅基于关键词分类不够，需要理解 issue 的完整上下文
- **修正方案**：分类时不仅看关键词，还要分析完整描述和期望行为
- **是否已纳入 pattern**：否 --- 此为通用性改进，已记入分类规则更新
