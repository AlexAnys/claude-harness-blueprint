# 每周代码质量报告

## 来源
团队希望每周一自动生成上周的代码质量趋势报告。

## 调度
- 频率：每周一 09:00
- 触发方式：`launchctl` (macOS) 或 `cron` (Linux) → `claude "执行 weekly-report"`

## 目标
生成一份结构化报告，包含：
1. 上周提交统计（commit 数、文件变更数、贡献者）
2. 测试覆盖率趋势
3. lint 警告数量变化
4. 新增 / 关闭的 issue 统计
5. 值得关注的变更摘要

## 步骤
1. 运行 `git log --since="7 days ago" --stat`
2. 运行测试套件，提取覆盖率数据
3. 运行 linter，统计警告数
4. 查询 GitHub issue 状态变化
5. 生成 Markdown 报告写入 `runs/YYYY-MM-DD.md`

## 成功标准
- 报告包含上述 5 个部分
- 数据来自实际命令输出，不是编造
- 与上周报告格式一致
- 3 分钟内完成

## 失败处理
- 命令执行失败：记录错误，跳过该部分，在报告中标注 `[数据获取失败]`
- 超时：记录已完成的部分，标注超时
